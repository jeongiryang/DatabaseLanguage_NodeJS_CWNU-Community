# AI 활용 명시 문서

## 1. 문서 목적

본 문서는 데이터베이스개론 과제 제출 시 `CWNU Community 게시판 시스템` 개발 과정에서 AI를 활용한 사실과 활용 범위를 명확히 밝히기 위한 문서입니다.

AI는 프로젝트 구현을 대신하는 주체가 아니라, 설계 검토, 구현 보조, 오류 분석, 문서화 정리에 활용한 보조 도구입니다. 최종 기능 검토와 제출물 구성 책임은 제출자에게 있습니다.

## 2. 사용한 AI 도구

| 도구 | 활용 목적 |
|---|---|
| OpenAI Codex | 코드 작성 보조, 파일 수정 보조, 테스트 체크리스트 정리 |
| ChatGPT | 구현 전략 정리, 설명 문서 작성 보조, 오류 원인 분석 보조 |

## 3. AI 활용 범위

| 활용 항목 | 활용 내용 |
|---|---|
| 프로젝트 구조 설계 보조 | Node.js + Express + Prisma 기반 폴더 구조와 기능 구현 순서 정리 |
| Express/Prisma 기반 API 구현 보조 | 인증, 게시글, 댓글, 반응, 마이페이지 관련 API 구현 방향 검토 |
| DB relation, cascade 삭제, unique 제약 검토 보조 | User, Post, Comment, Like, Dislike, Bookmark 관계와 삭제 정책 검토 |
| UI/UX 개선 보조 | 다크모드, hover 효과, footer, 도움말 투어, 반응 버튼 UI 개선 방향 정리 |
| 오류 원인 분석 및 해결 방향 제안 | 라우팅, 권한 검증, UI 예외 처리, Prisma 관련 문제 해결 방향 제안 |
| 회귀 테스트 체크리스트 작성 보조 | 회원가입, 로그인, 게시글, 댓글, 좋아요, 북마크, 회원 탈퇴 등 기능별 검증 항목 정리 |
| README 및 기능 설명서 작성 보조 | 과제 보고서형 README와 `docs/feature-guide.md` 문서 구조 정리 |
| Git 커밋 메시지와 작업 순서 정리 보조 | 기능 단위 작업 구분, 커밋 메시지 초안, 변경 파일 요약 정리 |

## 4. 사람이 직접 수행한 검토

| 검토 항목 | 직접 수행한 내용 |
|---|---|
| 브라우저 기능 확인 | 회원가입, 로그인, 게시글 작성/조회/수정/삭제, 댓글/답글, 반응 기능 등을 브라우저에서 확인 |
| Prisma Studio 확인 | DB에 저장된 사용자, 게시글, 댓글, 좋아요, 싫어요, 북마크 데이터 확인 |
| 터미널 검증 | `node --check`, `npx prisma validate`, `npm run dev` 등 실행 결과 확인 |
| Git 관리 | `git status`, commit, push를 직접 관리하고 변경 파일을 확인 |
| 민감정보 확인 | `.env`, DB 연결 문자열, `JWT_SECRET`이 문서와 커밋에 포함되지 않도록 확인 |
| 기능별 테스트 후 반영 | AI가 제안한 코드를 그대로 신뢰하지 않고 기능 단위로 동작을 확인한 뒤 반영 |

## 5. AI 활용 시 주의한 점

| 주의 항목 | 처리 내용 |
|---|---|
| `.env` 파일 미공개 | `.env` 파일은 문서에 작성하지 않고 커밋 대상에서 제외 |
| DB 연결 문자열 비공개 | `DATABASE_URL`, `DIRECT_URL`의 실제 값은 문서와 대화에 작성하지 않음 |
| JWT secret 비공개 | `JWT_SECRET` 실제 값은 문서와 커밋에 포함하지 않음 |
| 실제 사용자 개인정보 미사용 | seed 데이터에는 테스트용 이메일과 닉네임만 사용 |
| 과제 요구사항 검토 | AI 결과를 프로젝트 구조와 데이터베이스개론 과제 요구사항에 맞게 검토 |
| 기능 단위 작업 | 구현 단위를 기능별로 분리하여 오류 확산을 줄이고 회귀 테스트를 쉽게 수행 |

## 6. 최종 책임

AI는 본 프로젝트에서 보조 도구로 활용했습니다. AI가 제안한 코드, 문서, 테스트 항목은 제출자가 직접 검토하고 프로젝트 요구사항에 맞게 수정했습니다.

따라서 최종 제출물의 구성, 기능 동작 확인, 민감정보 제거, 과제 요구사항 충족 여부에 대한 책임은 제출자에게 있습니다.

## 7. 요약

본 프로젝트에서 AI는 단순 코드 생성 도구가 아니라 설계 검토, 구현 보조, 오류 분석, 테스트 체크리스트 작성, 문서화 보조 도구로 활용되었습니다.

AI 활용 결과는 최종 제출 전에 브라우저, 터미널, Prisma Studio, Git 상태 확인을 통해 검토했으며, 실제 환경변수와 민감정보는 문서에 포함하지 않았습니다.
